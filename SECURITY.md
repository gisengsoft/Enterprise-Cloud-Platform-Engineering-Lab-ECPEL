# Security Policy

## Scope

ECPEL is currently an educational and portfolio repository, not a production service. Repository content may describe target architecture, simulated labs, validation workflows, and operational practices, but those materials are not evidence of a deployed secure platform.

## Supported Content

Only content present in the default branch should be treated as current repository content. This policy does not create software-version support guarantees, service levels, or maintenance commitments.

## Reporting a Vulnerability

Do not post sensitive security findings publicly.

If GitHub private vulnerability reporting is available in the repository interface, use that option for sensitive reports. If private reporting is not available, open a minimal public issue that contains no sensitive details and request a private communication channel with the repository maintainers.

Do not include exploit details, credentials, secrets, customer data, personal information, or internal infrastructure details in a public issue.

## Sensitive Information

Do not submit:

- active credentials;
- secrets;
- tokens;
- private keys;
- exploitable customer data;
- personal information;
- internal infrastructure details.

If sensitive information is accidentally submitted, stop using the exposed value, rotate or revoke it where applicable, and avoid repeating the sensitive value in follow-up comments or commits.

## Response Expectations

Reports will be reviewed according to maintainer availability and repository scope. This policy does not promise specific response or remediation times.

## Security Boundaries

- Documentation is not proof of secure deployment.
- Simulated labs are not production security controls.
- No certification or compliance guarantee is provided.
- Cloud security must be validated in an authorized environment.

## Disclosure

Coordinated disclosure is encouraged so security issues can be reviewed responsibly and sensitive details can be limited. This policy does not create contractual, legal, service-level, or remediation commitments.
