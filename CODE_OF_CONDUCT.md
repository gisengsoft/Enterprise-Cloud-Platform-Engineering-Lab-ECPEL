# ECPEL Code of Conduct

## Purpose

ECPEL is intended to support careful, evidence-backed discussion about cloud platform engineering. This Code of Conduct defines expectations for constructive collaboration while preserving the repository principle: "No evidence, no implementation."

## Expected Behavior

Participants are expected to practice:

- respectful technical discussion;
- constructive feedback focused on the work;
- evidence-based disagreement;
- inclusive collaboration;
- privacy and security awareness;
- responsible use of automation and AI-generated contributions.

Automation-assisted contributions should be reviewed before submission. Contributors remain responsible for accuracy, licensing considerations, security implications, and evidence quality.

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment;
- discrimination;
- personal attacks;
- publishing private information;
- deliberate misinformation;
- fabricated evidence;
- unsafe disclosure of credentials or vulnerabilities;
- abusive or disruptive conduct.

Disagreement about technical direction is acceptable when it remains respectful, specific, and evidence-based.

## Scope

This policy applies to repository issues, Pull Requests, discussions, code review, documentation review, and other project-related communication channels controlled by the repository maintainers.

## Reporting

Use repository-maintainer channels to report conduct concerns. For security-sensitive matters, use GitHub private vulnerability reporting when that option is available. If private reporting is not available, open a minimal public issue without sensitive details and request a private communication channel.

Do not publish credentials, private information, vulnerability details, or other sensitive material while reporting a concern.

## Enforcement

Maintainers may take proportional actions based on context and severity, including:

- clarification;
- warning;
- content removal;
- contribution restriction;
- repository access restriction.

Enforcement decisions should aim to protect constructive collaboration, repository integrity, and participant safety. This document does not create legal claims, guarantees, or service commitments.
